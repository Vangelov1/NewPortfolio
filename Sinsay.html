UI/UX Design & Development

Project Brief & Analisis

There were several things that stood out at first glance that really bothered me when I first inspected the website layout and the structure of the current design. A lot of inconsistencies are clearly visible and hierarchy issues that show up.

Let's start with what we are greeted with. The first thing we see is the header navigation and the lower banner.

1.Home Page


The home page has some issues that can be seen at first glance. The main one being that some of the elements aren't following the layout grid, other elements are batteling for attention and need adjustments, for example the text block on the top containing the cookies disclaimer is doing just that with the banner.

The Solution

I ended up changing a lot of things in order to follow some simple layout rules that were broken. I had to re-do the entire bottom section including the footer, that was the most bothering section. I also removed the top two text blocks because they were taking attention away from the banner.

Nav Menu

The layout seemed a bit broken in hierarchy and it didn't feel clean to me so I made some minor changes to freshen it up.

Nav Solution

Search Bar

The search bar is a totall mess, it overflows on my laptop screen and has a huge battle going on in all areas of the screen.

Search Solution

I ended up remaking the entire thing and made it more simple, to reduce usage stress.

Login / Register Page

I really loved the login and register design, specifically that both login and register forms were on one page, where one half would hide and one would show when you simply click on the button, so I kept the design the same and made some minor alterations and changes so it would look a bit better.

Also, the paddings and margins were bugging me when I was looking at them so I wanted to re-do them as well.

Login/Register Solution

User CP

The User CP Layout is noticabley bad, I don't really need to comment on it, so lets move on to the new layout and solution to this problem.

User CP Solution

Cart Page

I changed the layout and took out some of the content that would be better displayed on the final order page. I remade the layout so I can add it to the user CP like a feature, just so it looks more complete because the current user CP looks like it's lacking.


Product Page

I didn't like how most of the content was placed on the left side of the page, also how there was too much text, so I took out some content, for example the Shipping and Return content. I decided that they should be on a dedicated page by themselves.

Product Page Solution

The solution was to rearrange some content and cut down on some info that wasn't really necessary.


Checkout

I didn't like the positioning so I changed some things.

Cjeckout Solution

Final Thoughts

The end result within our minds is a perfect blend of simplicity and functionality. We managed to create a tool that is simple in nature but provides a very powerful spectrum of resources that help any scale of user to manage the health of their online presence, from large corporations to everyday individuals. Uptime monitor was designed to be able to scale as much as it is needed to.
